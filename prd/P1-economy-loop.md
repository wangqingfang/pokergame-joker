# PRD-P1：经济闭环（钱包 + 入场费 + 输光补给 + 简易技能购买）

> 来源：`product_requirement_document_v2.md` § 七 落地优先级 — P1 阶段
> 状态：待排期
> 阶段价值：跑通"赢局攒币 → 入场消耗 → 输光救济 → 用币买技能"的最小经济闭环。

---

## 1. 目标与非目标

### 目标
- 玩家拥有跨局持久化的金币钱包。
- 每局必须支付入场费才能开始。
- 钱包不足时触发"24h 补给"机制（首 3 次冷却 4h，之后 24h）。
- 玩家可以用金币**直接列表购买**已有 6 个技能之外的扩展技能（非树状，单层列表）。

### 非目标（留给 P2/P3）
- 技能树 UI 与流派分化。
- 出战配置与 4 技能上限。
- 抽卡。

---

## 2. 用户故事

- 作为玩家，我希望赢的钱攒下来，输了能心疼，否则没动力赢。
- 作为玩家，我输光时不希望被永久卡死，但也不希望补给来得太轻松。
- 作为玩家，我希望能用攒的钱解锁原本没有的新技能。

---

## 3. 数值规格

| 项 | 值 |
|---|---|
| 初始钱包 | 2000 金币 |
| 入场费 | 500 金币 → 桌上 1000 筹码（2:1） |
| 输光阈值 | 钱包 < 500 |
| 补给金额 | 1500 金币 |
| 补给冷却 | 第 1/2/3 次 4h，第 4 次起 24h |
| 单局结算 | 桌上筹码 ÷ 2 → 加回钱包 |
| 扩展技能购买价（P1 列表版） | 800 金币 / 个（统一价） |

---

## 4. 功能需求

### 4.1 钱包持久化
- 使用 `UserDefaults` 存储 `Wallet` 结构：`coins`、`totalPlayMinutes`、`lastBailoutAt`、`bailoutCount`、`ownedExtraSkillIds`。
- 启动 App 时加载，每次变更立即写入。

### 4.2 入场费
- "开始一局"按钮显示"入场 500"。
- 钱包 ≥ 500 时可点击；否则按钮置灰，提示"金币不足，等待补给"。
- 点击后扣除 500，进入牌局，桌上初始筹码 1000。

### 4.3 输光补给
- 每手结束后判定钱包是否 < 500。
- 若是，弹出"补给窗"：
  - 倒计时（`lastBailoutAt + 冷却` − 当前时间）
  - 倒计时为 0 时显示"领取 1500 金币"按钮，点击后 `coins += 1500`、`bailoutCount += 1`、`lastBailoutAt = now`。
  - 未到时间显示倒计时，关闭按钮可以退出主菜单等。

### 4.4 简易技能商店
- 主菜单新增入口"技能商店"。
- 列表展示**P1 阶段开放的扩展技能**（建议先放 4 个，复用 v2 文档里 T1 蓝色的部分技能名）：
  - 重拳（暴力）：本手 raise 时金额 +50%
  - 迷雾（智谋）：让对手看不到你的下注金额 1 手
  - 回血（守护）：本手输时返还 30% 入场费
  - 标记（诡谲）：标记一名 AI，下次 showdown 知道其底牌
- 每张卡片显示名称、描述、价格、状态（未拥有 / 已拥有）。
- 点击购买扣金币，加入 `ownedExtraSkillIds`。

### 4.5 技能装载（P1 简化版）
- 已购扩展技能默认全部加入当局技能栏（不限数量），与原 6 个并列。
- 若超过界面容量，技能栏改用横向滚动。

## 5. 数据模型

```swift
struct Wallet: Codable {
    var coins: Int = 2000
    var totalPlayMinutes: Int = 0
    var lastBailoutAt: Date? = nil
    var bailoutCount: Int = 0
    var ownedExtraSkillIds: [String] = []
}

enum ExtraSkillId: String, CaseIterable {
    case bruteFist, mageMist, guardianHeal, tricksterMark
    var name: String { ... }
    var price: Int { 800 }
}
```

持久化层封装在 `WalletStore: ObservableObject`，通过 `@EnvironmentObject` 注入。

## 6. UI 改动清单

- **主菜单**：新增钱包条（顶部）、"技能商店"入口、"开始一局（入场 500）"按钮。
- **技能商店页**：列表 + 购买按钮 + Toast 反馈。
- **结算页**：钱包变化展示（+X / -500）。
- **补给弹窗**：倒计时 + 领取按钮。

---

## 7. 验收标准

- [ ] 钱包数值能跨 App 重启保持。
- [ ] 钱包 < 500 时无法开局。
- [ ] 补给冷却逻辑符合 4/4/4/24 规则。
- [ ] 购买技能后金币正确扣除，重启 App 后技能仍存在。
- [ ] 单局结算后钱包正确增减。

---

## 8. 风险与缓解

- **本地存档可被篡改**：MVP 不处理，注释写明"未来需账号化"。
- **补给被滥用（用户改系统时间）**：MVP 不处理，记录 `lastBailoutAt` 仅做轻量校验。
- **数值膨胀**：定期跑模拟，必要时下调单局结算系数。

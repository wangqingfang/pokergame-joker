# 需求文档

## 1. 应用概述

### 1.1 应用名称
无厘头德州扑克

### 1.2 应用描述
一款充满无厘头、搞笑风格的德州扑克网页游戏。玩家在标准德州扑克玩法基础上，可以使用各种搞笑技能影响游戏进程。游戏采用卡通/漫画风格UI，支持单人对战多个AI对手，每个AI具有不同的搞笑性格。

## 2. 用户与使用场景

### 2.1 目标用户
喜欢德州扑克和休闲娱乐的玩家，追求轻松搞笑的游戏体验

### 2.2 核心使用场景
玩家在休闲时间打开网页，与AI对手进行一局或多局德州扑克对战，通过使用搞笑技能增加游戏趣味性

## 3. 页面结构与功能说明

### 3.1 页面结构

```
无厘头德州扑克
├── 游戏主界面
│   ├── 牌桌区域
│   ├── 玩家区域
│   ├── AI对手区域
│   ├── 公共牌区域
│   ├── 底池显示
│   ├── 操作按钮区
│   └── 技能栏
└── 结算界面
```

### 3.2 游戏主界面

#### 3.2.1 牌桌区域
- 展示卡通/漫画风格的牌桌背景
- 显示当前游戏回合信息
- 展示搞笑特效和动画

#### 3.2.2 玩家区域
- 显示玩家的搞笑角色/头像（搞笑动物或表情包风格）
- 显示玩家手牌（2张）
- 显示玩家当前筹码数量
- 显示玩家当前状态（跟注、加注、弃牌等）

#### 3.2.3 AI对手区域
- 显示多个AI对手（建议3-5个）
- 每个AI显示搞笑角色/头像
- 显示AI手牌背面
- 显示AI当前筹码数量
- 显示AI当前状态
- AI具有不同的搞笑性格特征

#### 3.2.4 公共牌区域
- 按德州扑克规则依次展示公共牌
- 翻牌阶段：显示3张公共牌
- 转牌阶段：显示第4张公共牌
- 河牌阶段：显示第5张公共牌

#### 3.2.5 底池显示
- 显示当前底池筹码总数
- 显示各玩家本轮已下注筹码

#### 3.2.6 操作按钮区
- 跟注按钮：跟随当前最高下注额
- 加注按钮：增加下注额
- 弃牌按钮：放弃本局
- 过牌按钮：不下注继续游戏（当前无需跟注时可用）

#### 3.2.7 技能栏
- 显示玩家可用技能列表
- 每个技能显示名称、图标、冷却状态
- 技能包括但不限于：
  - 换牌术：强制交换一张牌
  - 偷看：偷看对手一张牌
  - 倒霉蛋：让对手下一轮必须跟注
  - 混乱：随机重新洗一张公共牌
  - 护盾：免疫一次技能攻击
  - 吃瓜：从牌堆顶摸一张替换最差的牌
- 点击技能图标使用技能
- 技能使用后进入冷却时间

### 3.3 结算界面

- 显示本局游戏结果
- 展示所有玩家的手牌
- 显示获胜者及获胜牌型
- 显示筹码变化
- 提供「继续游戏」按钮
- 提供「退出游戏」按钮

## 4. 业务规则与逻辑

### 4.1 德州扑克基础玩法规则

#### 4.1.1 发牌阶段
- 每位玩家（包括AI）发2张底牌
- 玩家可见自己的底牌，AI底牌对玩家不可见

#### 4.1.2 下注阶段
- 按顺序进行下注
- 玩家可选择跟注、加注、弃牌、过牌（无需跟注时）
- AI根据性格和牌力做出决策

#### 4.1.3 翻牌阶段
- 发出3张公共牌
- 进行新一轮下注

#### 4.1.4 转牌阶段
- 发出第4张公共牌
- 进行新一轮下注

#### 4.1.5 河牌阶段
- 发出第5张公共牌
- 进行最后一轮下注

#### 4.1.6 比牌阶段
- 未弃牌的玩家亮出底牌
- 根据德州扑克牌型大小规则判定胜负
- 牌型从大到小：皇家同花顺、同花顺、四条、葫芦、同花、顺子、三条、两对、一对、高牌
- 获胜者获得底池所有筹码

### 4.2 技能系统规则

#### 4.2.1 技能使用时机
- 技能可在玩家回合时使用
- 部分技能可在特定阶段使用（如翻牌前、翻牌后）

#### 4.2.2 技能效果
- 换牌术：选择自己一张手牌，从牌堆随机抽取一张替换
- 偷看：选择一个对手，查看其一张底牌
- 倒霉蛋：选择一个对手，该对手下一轮必须跟注（不能弃牌）
- 混乱：随机选择一张已发出的公共牌，重新从牌堆抽取替换
- 护盾：激活后免疫下一次针对自己的技能攻击
- 吃瓜：从牌堆顶摸一张牌，可选择替换手牌中较差的一张

#### 4.2.3 技能冷却
- 每个技能使用后进入冷却时间
- 冷却时间以回合数计算
- 冷却期间技能不可使用

#### 4.2.4 技能互动
- 护盾可以抵消针对自己的技能效果
- AI也可以使用技能

### 4.3 AI性格系统

- 每个AI具有不同的搞笑性格特征
- 性格影响AI的下注策略和技能使用倾向
- 性格示例：激进型、保守型、随机型、搞怪型等

### 4.4 游戏结束条件

- 当只剩一位玩家拥有筹码时，游戏结束
- 玩家筹码为0时，游戏结束
- 玩家可随时选择退出游戏

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 玩家筹码不足以跟注 | 只能选择弃牌或全下（All-in） |
| 技能冷却中被点击 | 提示技能冷却中，无法使用 |
| 使用护盾时无技能攻击 | 护盾效果保留至下次受到技能攻击 |
| 混乱技能在翻牌前使用 | 提示需要在公共牌发出后使用 |
| 偷看技能目标已弃牌 | 提示该对手已弃牌，无法偷看 |
| 换牌术/吃瓜时牌堆无牌 | 提示牌堆已空，技能无法使用 |
| 所有AI同时弃牌 | 玩家直接获胜，进入结算 |
| 多人同时All-in | 按标准德州扑克规则处理边池 |

## 6. 验收标准

1. 玩家进入游戏主界面，看到自己和多个AI对手的角色头像及初始筹码
2. 游戏发牌，玩家收到2张底牌，AI收到底牌（背面显示）
3. 玩家进行下注操作（跟注/加注/弃牌/过牌），AI依次做出决策
4. 游戏依次展示翻牌（3张）、转牌（1张）、河牌（1张），每个阶段进行下注
5. 玩家在自己回合使用一个技能（如「偷看」），成功查看对手一张底牌
6. 进入比牌阶段，系统判定牌型大小，显示获胜者
7. 进入结算界面，显示本局结果和筹码变化
8. 玩家点击「继续游戏」，开始新一局

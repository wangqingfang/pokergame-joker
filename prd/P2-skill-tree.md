# PRD-P2：技能树 UI + 4 流派 + 升级系统

> 来源：`product_requirement_document_v2.md` § 七 落地优先级 — P2 阶段
> 前置：P1 已上线（钱包、入场费、技能商店列表）
> 阶段价值：把 P1 的"列表式商店"升级为四流派技能树，建立 build 多样性与长线消耗点。

---

## 1. 目标与非目标

### 目标
- 把扩展技能按"暴力 / 智谋 / 守护 / 诡谲"四流派组织，每派 9 节点（3 层 × 3 分支）。
- 节点解锁需同时满足：游戏时长 + 前置节点 + 金币。
- 已购技能可升级 5 级，每级 +6% 成功率。
- 技能成功率纳入实际牌局判定（之前是 100% 触发）。

### 非目标（留给 P3/P4）
- 出战配置（仍延续 P1 的"全部装载"，4 技能上限留给 P3）。
- T3 橙卡与抽卡。
- AI 流派人设。

---

## 2. 流派与节点表

四流派各 9 节点，用统一结构定义：

```
T1 (蓝, 60% 基础) — 3 节点，无前置，门槛 0h，单价 800
T2 (紫, 45% 基础) — 3 节点，前置任一 T1，门槛 2h，单价 3000
T3 (橙, 30% 基础) — 1 节点（旗舰），P2 阶段仅占位锁定，待 P4 抽卡解锁
```

### 暴力流（红 / Brute）
- T1：重拳 / 震慑 / 挑衅
- T2：连续重拳 / 全场翻倍盲 / 强制跟注
- T3（占位）：撕牌

### 智谋流（蓝 / Mage）
- T1：偷看 / 换牌 / 迷雾
- T2：双偷看 / 改公共牌 / 概率扭曲
- T3（占位）：操控全场

### 守护流（绿 / Guardian）
- T1：护盾 / 回血 / 反弹
- T2：双护盾 / 团队加持 / 入场费返还
- T3（占位）：免疫一手

### 诡谲流（紫 / Trickster）
- T1：慢毒 / 标记 / 隐身
- T2：组合阴招 / 伪装手牌 / 记号牌
- T3（占位）：偷天换日

> 命名沿用 v2 PRD § 三，可在排期时进一步细化文案。

## 3. 升级与概率

| 层级 | 基础成功率 | 满级（+5×6%）| 升级累计成本 |
|---|---|---|---|
| T1 | 60% | 90% | 200 / 400 / 800 / 1600 / 3200 |
| T2 | 45% | 75% | 800 / 1600 / 3200 / 6400 / 12800 |
| T3 | 30% | 60% | 仅碎片合成（P4） |

技能释放时 roll 一个 [0,1) 随机数，< 当前成功率则生效，否则飘字提示"翻车"并消耗冷却。

## 4. 数据模型扩展

```swift
enum School: String, Codable { case brute, mage, guardian, trickster }
enum SkillTier: Int, Codable { case t1 = 1, t2 = 2, t3 = 3 }

struct SkillNode: Identifiable, Codable {
    let id: String
    let name: String
    let school: School
    let tier: SkillTier
    let baseSuccess: Double          // 0.6 / 0.45 / 0.3
    let unlockMinutes: Int           // 0 / 120 / 480
    let buyCost: Int                 // 800 / 3000 / 0
    let prerequisiteIds: [String]    // T1 为空；T2 为同流派任一 T1
    var owned: Bool = false
    var level: Int = 0               // 0..5
    var currentSuccess: Double { baseSuccess + Double(level) * 0.06 }
}

struct SkillTree: Codable {
    var nodes: [SkillNode]   // 36 个节点（4 派 × 9）
}
```

`SkillTreeStore: ObservableObject` 负责持久化、解锁判定、升级扣费。

## 5. UI 设计

### 5.1 技能树页（新）
- 顶部：4 流派 Tab + 钱包余额 + 累计时长。
- 主区：3 列 × 3 行节点矩阵（T1 在底，T2 中，T3 顶，模拟"向上成长"）。
- 节点状态：
  - 灰底：未达解锁条件
  - 蓝边：可购买
  - 金边：已拥有（显示等级 ★×N）
  - 锁图标：T3 阶段统一显示"待抽卡解锁"
- 点节点弹详情卡：名称、描述、流派色、价格、成功率曲线（基础 → 满级）、按钮（购买 / 升级 / 锁定原因）。

### 5.2 技能库页（新）
- 入口：主菜单"我的技能"。
- 网格列出已购技能，按流派分 Tab。
- 点项目可一键跳转到技能树对应节点。

### 5.3 主菜单更新
原 P1 的"技能商店"入口改名为"技能树"，并新增"我的技能"。

## 6. 牌局逻辑改动

- 释放技能时调用 `SkillNode.currentSuccess` 计算成败：
  - 成功 → 原效果 + 飘字 "✨ 命中"
  - 失败 → 不生效 + 飘字 "💥 翻车" + 仍进入冷却
- 失败也算释放，**不退还任何资源**。

---

## 7. 验收标准

- [ ] 4 流派各 9 节点完整可见，T1 默认可购，T2 受时长门槛限制。
- [ ] 升级到 5 级后按钮置灰，显示"已满级"。
- [ ] 实际牌局中翻车概率与 UI 显示的成功率一致（可通过 100 次模拟验证）。
- [ ] 关 App 再开，已购节点与等级保留。

## 8. 风险与缓解

- **节点描述与现有 6 个基础技能重名**：P2 阶段把"扩展技能"和"基础 6 技能"独立命名，技能树只管扩展技能。
- **数值膨胀**：T2 满级累计成本接近 25k，刚好对齐"中后期目标"。需要在 P1 上线后跟一次实际经济模拟再定稿。
- **UI 复杂度**：树形布局用静态网格而非自由树，降低前端工作量。
